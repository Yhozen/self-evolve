# Target State

This page summarizes the intended end state of the repository's operating model.

## Intended Capabilities

The system should eventually be able to:

- understand the current goal and repository context
- choose the next smallest meaningful improvement
- work in an isolated worktree
- verify and evaluate its own output
- create one or more reviewable pull requests
- decompose larger work into stacked pull requests
- ask the human for missing context or difficult decisions
- discard weak or low-confidence runs

## Placeholder Areas

The details below are intentionally not concrete yet:

- the first real project goal
- domain-specific architecture
- concrete run blueprints tied to real tooling
- evaluation thresholds backed by automated checks

These should be filled as the repository is used for a concrete goal.
