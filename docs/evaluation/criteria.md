# Evaluation Criteria

Before keeping a run, ask:

- Is the change aligned with the goal and current project direction?
- Is the change scoped tightly enough to review confidently?
- Were available checks run, or was the absence of checks stated clearly?
- Does the reasoning and documentation make the result understandable to a human reviewer?
- Does the result improve the repository enough to justify keeping it?

## Expected Evidence

Each substantial run should ideally leave behind:

- a concise statement of the task or objective
- the files changed
- the checks that were run, or the reason checks were unavailable
- any important trade-offs or open questions
- a keep or discard recommendation

The repository does not yet have concrete verification commands or domain-specific acceptance thresholds.
