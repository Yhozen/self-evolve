# Glossary

- **Blueprint** - A reusable workflow that defines how a type of autonomous run should operate.
- **Canonical page** - A durable wiki page that summarizes the current understanding of some topic.
- **Discarded run** - A run that produced low-confidence or low-value output and should not be proposed as accepted work.
- **Docs wiki** - The persistent knowledge layer under `docs/`.
- **Goal** - The product or business objective the repository is evolving toward.
- **Placeholder** - A clearly marked section or page that exists in its final location before concrete content is available.
- **Run artifact** - A historical record of a specific run, such as a brief, evaluation, or outcome note.
- **Schema** - The instructions that tell the agent how to maintain the wiki and execute workflows.
